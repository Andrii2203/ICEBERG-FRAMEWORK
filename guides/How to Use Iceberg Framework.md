How to Use Iceberg Framework
Version: v1.0
Purpose: Explain how to operate Iceberg Framework in real projects.

0. Introduction
Iceberg Framework — це система стандартів, протоколів, шаблонів і гайдів, створена для того, щоб:

робити міграції передбачуваними

будувати нові сайти структуровано

забезпечувати 1‑to‑1 відповідність поведінки

дозволяти AI та людям працювати разом без хаосу

гарантувати чисту архітектуру та відсутність legacy‑хаосу

Iceberg — це не бібліотека, не набір компонентів і не фреймворк у технічному сенсі.
Це процес, правила, контракти, артефакти і дисципліна.

2. How Iceberg Works
Iceberg працює за принципом:

Код
ANALYSIS → ARCHITECTURE → PLANNING → EXECUTION → VALIDATION → DELIVERY
2.1. Analysis
Ти не пишеш код, поки не зрозумів систему.
Артефакти: PageMapping, DeepAnalysis, FileAnalysis.

2.2. Architecture
Ти не пишеш код, поки не затверджена архітектура.
Артефакт: Architecture.md..

2.3. Planning
Ти не пишеш код, поки не створено ExecutionPlan.md..
План = покрокова інструкція.

2.4. Execution
Тільки після плану — реалізація.

2.5. Validation
Кожен крок перевіряється Validation Protocol.

2.6. Delivery
Фінальний пакет документів + код.

3. Rules of Using Iceberg
Zero Assumptions — якщо щось незрозуміло → STOP.

Determinism — однакові входи → однаковий результат.

No Magic — жодних прихованих рішень.

Architecture‑First — код не пишеться без архітектури.

Documentation‑Driven — кожен артефакт створюється до коду.

STOP‑CHECK — критичні точки зупинки.

Traceability — кожне рішення має бути пояснене.
