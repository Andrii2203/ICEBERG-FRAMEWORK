# ICEBERG FRAMEWORK — BRAND BOOK (v2)
A deterministic identity system for the Iceberg Framework.  
Built on structure, minimalism, semantic depth, and iceberg logic.

---

# 1. LOGO SYSTEM

## 1.1 Primary Logo
- Geometric, faceted iceberg  
- Visible tip: light blue + white  
- Submerged mass: dark blue facets  
- Typography: ICEBERG FRAMEWORK (uppercase, sans‑serif)  
- Usage: hero sections, official pages, presentations  

## 1.2 Monochrome Variants
- White on black  
- Black on white  
- Usage: documentation, watermark, dark/light UI  

## 1.3 Rules
- One logo form; variants = contrast adaptations  
- No distortion, recoloring, rotation, shadows  
- Minimum clear space: 1× iceberg height  
- Minimum size: 32px height

# 3. SEMANTIC COLOR MAPPING (ICEBERG LOGIC)
*The logo is not decorative. Every color has meaning.*

## 3.1 Submerged Structure (≈80%)

### Deep Ice Blue — #0A1A2F
Represents:  
- hidden logic  
- protocols  
- standards  
- STOP‑CHECK  
- persistent memory  
- determinism  
- the “real mass” of the framework  

### Arctic Blue — #1E3A5F
Represents:  
- structural facets  
- architecture  
- component hierarchy  
- technical geometry  

---

## 3.2 Visible Tip (≈20%)

### Iceberg White — #F8FAFC
Represents:  
- UI  
- documentation  
- visible behavior  
- the part users interact with  

---

## 3.3 Waterline / Separation Layer

### Frozen Gray — #CBD5E1
Represents:  
- boundary between visible and invisible  
- separation of UI and logic  
- structural precision  
- non‑decorative geometry  

---

## 3.4 Why Accent Colors Are NOT in the Logo

### Glacier Cyan — #38BDF8
- Reserved for **interaction** (links, hover)  
- Not part of symbolic identity  

### Steel Blue — #475569
- Reserved for **textual hierarchy**  

### Polar Night — #020617
- Reserved for **backgrounds**, not the symbol  

**Rule:**  
Accents = UI behavior  
Primary palette = brand identity
