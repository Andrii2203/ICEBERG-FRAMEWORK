## ❄ What Is Iceberg?
Iceberg Framework — це система документів, протоколів і стандартів, які забезпечують:

- передбачуваність  
- детермінізм  
- структурованість  
- відтворюваність 
- контрольованість  
- прозорість  

Iceberg не є бібліотекою або кодовим фреймворком.  
Це **операційна система для роботи з AI‑агентами**, яка визначає:

- як AI повинен думати  
- як AI повинен планувати  
- як AI повинен виконувати  
- як AI повинен документувати  
- як AI повинен взаємодіяти з людиною  
- як AI повинен будувати проєкти

## ❄ Core Philosophy
1. **Determinism**  
   Same inputs → same outputs.  
   Ніякої імпровізації. Ніяких припущень.

2. **Structure Over Creativity**  
   Спочатку структура → потім реалізація.

3. **Explicitness**  
   Кожен крок, кожне рішення, кожен артефакт — задокументований.

4. **Idempotency**  
   Кожен крок можна виконати повторно без побічних ефектів.

5. **STOP‑CHECK Governance**  
   AI зупиняється при найменшій невизначеності.

6. **Single Source of Truth**  
   Стандарти → Протоколи → Карти → Документація → Код.

## ❄ Components of Iceberg

### **1. Standards** → `/standards`
Описують *як має бути правильно*.  
Містять правила для:
- архітектури  
- якості  
- SEO  
- PWA  
- API  
- документація
- генерації проєктів  

---

### **2. Protocols** → `/core`
Описують *як AI повинен діяти*.  
Містять:
- STOP‑CHECK  
- правила поведінки  
- правила памʼяті  
- правила планування  
- правила міграції  
- правила QA
- Definition of Done  

---

### **3. Execution Maps** → `/core`
Описують *маршрут виконання*.  
Наприклад:
- Card 1 — створення нового сайту  
- Card 2 — міграція  
- Card 3 — рефакторинг  

---

## ❄ When to Use Iceberg
Iceberg Framework застосовується коли:

- потрібно створити новий проєкт з нуля  
- потрібно провести міграцію legacy → Next.js  
- потрібно привести існуючий репозиторій до ідеалу  
- потрібно працювати з AI‑агентами без хаосу  
- потрібна детермінована, відтворювана система  

---

## ❄ How Iceberg Works
1. AI активує Execution Map  
2. Створює памʼять (ICEBERG_TASK_DRAFT.md)  
3. Читає стандарти  
4. Читає протоколи  
5. Створює документацію  
6. Створює план  
7. Виконує план  
8. Проходить QA  
9. Завершує проєкт  

Усе — строго, покроково, без відхилень.

---

## ❄ Why Iceberg Exists
Щоб вирішити головні проблеми AI‑розробки:

- хаос  
- непередбачуваність  
- відсутність структури  
- відсутність контролю  
- відсутність відтворюваності  
- відсутність стандартів  

Iceberg — це відповідь.
