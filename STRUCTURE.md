# STRUCTURE.md — Iceberg Framework  
Official directory structure and component definitions for the Iceberg ecosystem.

Iceberg Framework consists of three layers:

1. **Iceberg Method** — proprietary methodology  
2. **Iceberg Framework (MIT)** — open‑source standards, protocols, templates  
3. **Iceberg Enterprise Engines** — proprietary commercial modules  

This document explains the purpose of each folder and file in the repository.

# 9. STRUCTURE PRINCIPLES

1. **Determinism** — no improvisation in file placement.  
2. **Separation of concerns** — method, standards, templates, guides, enterprise modules.  
3. **Dual‑license clarity** — open‑source vs proprietary clearly separated.  
4. **Scalability** — structure supports future modules.  
5. **Enterprise readiness** — logs, protocols, standards, engines.
